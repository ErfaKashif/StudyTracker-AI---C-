THEORY OF AUTOMATA - COMPREHENSIVE NOTES
=========================================

1. INTRODUCTION TO AUTOMATA THEORY
-----------------------------------
Automata Theory: Study of abstract machines and computational problems

Key Concepts:
- Formal Languages
- Grammars
- Automata (Abstract machines)
- Computability

Applications:
- Compiler design
- Text processing
- Pattern matching
- Protocol verification
- DNA sequencing
- Natural language processing

2. BASIC TERMINOLOGY
--------------------
Alphabet (Σ): Finite set of symbols
Example: Σ = {0, 1}, Σ = {a, b, c}

String: Finite sequence of symbols from alphabet
Example: 0110, abc, ε (empty string)

Language: Set of strings over an alphabet
- Can be finite or infinite
- Empty language: ∅
- Language containing empty string: {ε}

Operations on Strings:
- Length: |w| = number of symbols
- Concatenation: xy (join strings)
- Power: w^n (w repeated n times)
- Reverse: w^R (mirror of w)

Operations on Languages:
- Union: L1 ∪ L2
- Concatenation: L1 · L2 = {xy | x ∈ L1, y ∈ L2}
- Kleene Star: L* = {ε} ∪ L ∪ L² ∪ L³ ∪ ...
- Kleene Plus: L+ = L ∪ L² ∪ L³ ∪ ... (excludes ε)

3. FINITE AUTOMATA (FA)
-----------------------
Finite Automaton: Abstract machine with finite states

Types:
a) Deterministic Finite Automaton (DFA)
b) Non-deterministic Finite Automaton (NFA)

4. DETERMINISTIC FINITE AUTOMATON (DFA)
----------------------------------------
Definition: M = (Q, Σ, δ, q0, F)
- Q: Finite set of states
- Σ: Input alphabet
- δ: Transition function Q × Σ → Q
- q0: Initial state (q0 ∈ Q)
- F: Set of final/accepting states (F ⊆ Q)

Properties:
- Exactly one transition per state-symbol pair
- Deterministic: No ambiguity
- For every input, unique path exists

Example DFA:
States: {q0, q1}
Alphabet: {0, 1}
Accepts strings ending with '1'

Transition Table:
     | 0    | 1
-----|------|-----
→q0  | q0   | q1
*q1  | q0   | q1

Language Accepted: All strings ending with 1

DFA Construction Steps:
1. Identify language pattern
2. Determine minimum states needed
3. Define transitions for each state-symbol
4. Mark final states
5. Test with sample strings

5. NON-DETERMINISTIC FINITE AUTOMATON (NFA)
--------------------------------------------
Definition: M = (Q, Σ, δ, q0, F)
- δ: Transition function Q × (Σ ∪ {ε}) → 2^Q

Properties:
- Multiple transitions possible
- ε-transitions allowed (move without input)
- Non-deterministic: Multiple paths possible

Differences from DFA:
- Can have 0, 1, or more transitions per state-symbol
- May have ε-transitions
- Accepts if ANY path leads to final state

NFA to DFA Conversion (Subset Construction):
1. Create state for each subset of NFA states
2. Include ε-closure
3. Define transitions for each subset
4. Mark final states (contain NFA final state)

Result: DFA may have exponentially more states

6. REGULAR EXPRESSIONS (RE)
---------------------------
Regular Expression: Pattern describing regular language

Basic Operations:
- Union: r1 + r2 (or r1 | r2)
- Concatenation: r1 · r2 (or r1r2)
- Kleene Star: r*

Precedence: * > · > +

Examples:
- (0+1)*: All binary strings
- (a+b)*abb: Strings ending with 'abb'
- (0+ε)(1+ε): {ε, 0, 1, 01}
- a*b*: 0 or more a's followed by 0 or more b's

Regular Expression to NFA:
Thompson's Construction:
1. Basic NFAs for single symbols
2. Union: Add start state with ε-transitions
3. Concatenation: Connect NFAs
4. Kleene Star: Add loops

7. REGULAR LANGUAGES
--------------------
Regular Language: Language accepted by FA

Closure Properties (Regular languages closed under):
- Union
- Concatenation
- Kleene Star
- Complement
- Intersection
- Difference
- Reversal
- Homomorphism

Equivalence:
DFA ≡ NFA ≡ Regular Expression ≡ Regular Grammar

Pumping Lemma for Regular Languages:
If L is regular, then ∃ p (pumping length) such that
∀ string w ∈ L where |w| ≥ p,
w can be divided into xyz where:
1. |xy| ≤ p
2. |y| > 0
3. xy^i z ∈ L for all i ≥ 0

Use: Prove language is NOT regular

Example Non-Regular Languages:
- {0^n 1^n | n ≥ 0} (equal 0s and 1s)
- {ww | w ∈ {0,1}*} (string followed by itself)

8. CONTEXT-FREE GRAMMAR (CFG)
------------------------------
Definition: G = (V, Σ, P, S)
- V: Variables (non-terminals)
- Σ: Terminal symbols
- P: Production rules
- S: Start symbol

Production Rule Format:
A → α
where A ∈ V and α ∈ (V ∪ Σ)*

Example CFG:
S → aSb | ε
Generates: {a^n b^n | n ≥ 0}

Derivation:
- Leftmost: Replace leftmost variable first
- Rightmost: Replace rightmost variable first

Parse Tree:
- Root: Start symbol
- Leaves: Terminal symbols
- Internal nodes: Variables

Ambiguity:
Grammar is ambiguous if a string has:
- Multiple parse trees, OR
- Multiple leftmost/rightmost derivations

9. CONTEXT-FREE LANGUAGES (CFL)
-------------------------------
Language generated by CFG

Examples:
- Balanced parentheses
- Palindromes
- Arithmetic expressions
- Programming language syntax

Closure Properties:
Closed under:
- Union
- Concatenation
- Kleene Star
- Reversal

NOT closed under:
- Intersection
- Complement

Pumping Lemma for CFL:
If L is CFL, ∃ p such that ∀ w ∈ L where |w| ≥ p,
w = uvxyz where:
1. |vxy| ≤ p
2. |vy| > 0
3. uv^i xy^i z ∈ L for all i ≥ 0

Use: Prove language is NOT context-free

Example Non-CFL:
{a^n b^n c^n | n ≥ 0}

10. PUSHDOWN AUTOMATON (PDA)
-----------------------------
Automaton with:
- Finite states
- Input tape
- Stack (infinite memory)

Definition: M = (Q, Σ, Γ, δ, q0, Z0, F)
- Q: States
- Σ: Input alphabet
- Γ: Stack alphabet
- δ: Transition function Q × (Σ∪{ε}) × Γ → 2^(Q × Γ*)
- q0: Initial state
- Z0: Initial stack symbol
- F: Final states

Acceptance:
- By final state: End in accepting state
- By empty stack: Stack becomes empty

PDA Types:
a) Deterministic PDA (DPDA): At most one transition
b) Non-deterministic PDA (NPDA): Multiple transitions possible

Power: PDA ≡ CFG

CFG to PDA Conversion:
- Push start symbol
- For each production A → α, pop A and push α (reverse)
- Match terminals with input

11. TURING MACHINE (TM)
-----------------------
Most powerful computational model

Definition: M = (Q, Σ, Γ, δ, q0, B, F)
- Q: Finite states
- Σ: Input alphabet
- Γ: Tape alphabet (Σ ⊂ Γ)
- δ: Transition function Q × Γ → Q × Γ × {L, R}
- q0: Initial state
- B: Blank symbol
- F: Final states

Components:
- Infinite tape (both directions)
- Read/write head
- Finite control

Operations:
- Read symbol
- Write symbol
- Move head (Left or Right)

TM Types:
a) Deterministic TM: Single transition
b) Non-deterministic TM: Multiple transitions
c) Multi-tape TM: Multiple tapes
d) Universal TM: Simulates any TM

Acceptance:
- Halt in accepting state
- May loop forever (not accept)
- May crash (not accept)

12. CHOMSKY HIERARCHY
---------------------
Type 0: Recursively Enumerable (RE)
- Recognized by: Turing Machine
- Grammar: Unrestricted
- Production: α → β (no restriction)
- Most powerful

Type 1: Context-Sensitive Language (CSL)
- Recognized by: Linear Bounded Automaton
- Grammar: Context-sensitive
- Production: αAβ → αγβ where |γ| ≥ 1
- Length non-decreasing

Type 2: Context-Free Language (CFL)
- Recognized by: Pushdown Automaton
- Grammar: Context-free
- Production: A → γ

Type 3: Regular Language
- Recognized by: Finite Automaton
- Grammar: Regular
- Production: A → aB or A → a
- Least powerful

Containment:
Regular ⊂ CFL ⊂ CSL ⊂ RE

13. DECIDABILITY AND COMPUTABILITY
-----------------------------------
Decidable Problem: Algorithm exists that always halts with yes/no

Undecidable Problem: No algorithm can solve for all inputs

Famous Undecidable Problems:
1. Halting Problem: Does TM halt on input w?
2. Post Correspondence Problem (PCP)
3. Ambiguity of CFG
4. Equivalence of two CFGs

Church-Turing Thesis:
Any effectively computable function can be computed by TM

Recursive Language:
- Decidable
- TM halts on all inputs

Recursively Enumerable Language:
- Semi-decidable
- TM halts and accepts valid inputs
- May loop on invalid inputs

14. COMPLEXITY CLASSES
----------------------
P (Polynomial Time):
- Problems solvable in polynomial time
- Deterministic TM

NP (Nondeterministic Polynomial):
- Problems verifiable in polynomial time
- Non-deterministic TM

NP-Complete:
- Hardest problems in NP
- If one is in P, all NP is in P
- Examples: SAT, Vertex Cover, Hamiltonian Path

NP-Hard:
- At least as hard as NP-complete
- May not be in NP

P vs NP Question:
Open problem: Is P = NP?

15. MINIMIZATION OF DFA
-----------------------
Goal: Reduce states while accepting same language

Method (Table-Filling Algorithm):
1. Create table of state pairs
2. Mark pairs where one is final, other is non-final
3. Mark pairs that lead to marked pairs
4. Unmarked pairs are equivalent
5. Merge equivalent states

Myhill-Nerode Theorem:
Language is regular iff it has finite index

16. REGULAR GRAMMAR
-------------------
Types:
a) Right-Linear Grammar:
   A → wB or A → w

b) Left-Linear Grammar:
   A → Bw or A → w

Regular Grammar ≡ Regular Language ≡ FA

Conversion:
- Grammar to NFA
- NFA to Grammar

17. CONTEXT-SENSITIVE GRAMMAR
------------------------------
Production: α → β where |α| ≤ |β|
Exception: S → ε if S doesn't appear on RHS

Linear Bounded Automaton (LBA):
- TM with limited tape
- Tape length = constant × input length

18. APPLICATIONS
----------------
1. Compiler Design:
   - Lexical analysis: DFA
   - Syntax analysis: PDA/CFG
   - Parsing: Push-down automata

2. Text Editors:
   - Pattern matching: Regular expressions
   - Search/replace: Finite automata

3. Network Protocols:
   - Protocol verification: Automata
   - State machines: DFA/NFA

4. Natural Language Processing:
   - Grammar checking: CFG
   - Parsing: PDA

5. Computational Biology:
   - DNA sequencing: Pattern matching
   - Protein folding: Automata models

KEY FORMULAS AND RULES:
- DFA states = NFA states: 2^n states maximum
- Regular closure under all Boolean operations
- CFL not closed under intersection/complement
- Pumping lemma for proving non-regularity
- Chomsky hierarchy: Regular ⊂ CFL ⊂ CSL ⊂ RE

EXAM TIPS:
- Draw state diagrams clearly
- Check all transitions defined
- Test strings before finalizing
- Use pumping lemma systematically
- Remember closure properties
- Practice conversions (RE ↔ NFA ↔ DFA)
- Understand grammar derivations
